## Learning objective

Students will be able to define an algorithm, read and draw a flowchart, and trace the execution of common control-flow structures.

## Content stub

- **What is an algorithm:** a finite, ordered set of steps to solve a problem; correctness, efficiency, termination
- **Flowchart symbols:** oval (start/end), rectangle (process), diamond (decision), parallelogram (I/O); drawing conventions
- **Sequence:** statements execute top to bottom; most basic control flow
- **Selection:** if/else, switch/case; decision branching; truth tables for and/or/not
- **Iteration:** while, for, do-while loops; loop counter, condition, termination; infinite loop risk
- **Pseudocode:** language-agnostic algorithm description; bridge between English and code

## Assessment stub

- Flowchart reading: trace through a provided flowchart with specific input values and state the output
- Flowchart drawing: draw a flowchart for the algorithm 'find the largest number in a list of 5 numbers'
- Short answer: what makes an algorithm more efficient — fewer steps or faster steps?
