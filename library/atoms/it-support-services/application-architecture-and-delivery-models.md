## Learning objective

Students will be able to distinguish client-server, web, and cloud application architectures, and compare IaaS, PaaS, and SaaS delivery models.

## Content stub

- **Client-server architecture:** client sends requests; server processes and responds; two-tier vs. three-tier; middleware
- **Web applications:** browser as client; HTTP/HTTPS; stateless vs. stateful; front-end vs. back-end; APIs
- **SaaS:** software fully hosted and managed by vendor; user accesses via browser; examples: Salesforce, Gmail
- **PaaS:** provider manages OS and runtime; developer deploys code; examples: Heroku, Azure App Service
- **IaaS:** provider supplies virtual hardware; customer manages OS up; examples: AWS EC2, Azure VMs
- **Hybrid and multi-cloud:** mixing cloud providers and on-premises; why organizations avoid single-vendor lock-in

## Assessment stub

- Model classification: given 8 services, identify each as SaaS/PaaS/IaaS
- Architecture diagram: draw a three-tier web app showing browser, web server, app server, and database
- Short answer: why would a startup choose SaaS email over running their own mail server?
