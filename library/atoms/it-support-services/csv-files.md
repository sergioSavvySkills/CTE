## Learning objective

Students will be able to explain the CSV format, open and edit CSV files in a spreadsheet application, and describe common IT use cases for CSV data interchange.

## Content stub

- **What is a CSV:** comma-separated values; plain text; no formatting; universal compatibility
- **Opening CSV in a spreadsheet:** Text Import Wizard; delimiter selection; handling quoted fields with commas
- **Editing and saving:** modifying data; saving back as .csv vs. converting to .xlsx; encoding (UTF-8 vs. ANSI)
- **Common IT uses:** exporting user lists from AD, import/export of ticketing data, network device configs, log analysis
- **CSV limitations:** no formulas, no formatting, no multiple sheets; single data table only
- **Other delimited formats:** TSV (tab-separated), pipe-separated; when to use each

## Assessment stub

- CSV import lab: open a provided CSV file in Excel/Sheets, clean up formatting issues, and save as .xlsx
- IT use case: explain the steps to export an Active Directory user list to CSV and open it in Excel
- Short answer: what problem can arise if a CSV field contains a comma, and how is it handled?
