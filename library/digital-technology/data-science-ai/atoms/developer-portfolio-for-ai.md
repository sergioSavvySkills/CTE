## Learning objective

Students will be able to set up a public developer portfolio that showcases AI/DS projects.

## Content stub

- **GitHub as portfolio:** pin 6 best repos, curate the profile README, keep repos well-documented
- **Project README essentials:** problem, approach, results, how to run, screenshots, link to live demo if any
- **Live demos:** Hugging Face Spaces (free), Streamlit Community Cloud (free), GitHub Pages for static
- **Personal website:** short bio, project cards with thumbnails, contact info, resume link
- **Blog posts or dev logs:** write about what you built and what you learned; Medium, Dev.to, or your own site
- **Signals employers look for:** shipping real things, debugging breadth, clear writing, public feedback incorporation

## Assessment stub

- Lab: create your GitHub profile README with a bio and 3 pinned project ideas
- Critique: review a professional portfolio and identify what makes it effective
- Short answer: why does 'number of repos' matter less than '3 polished repos'?
