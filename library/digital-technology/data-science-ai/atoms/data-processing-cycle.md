## Learning objective

Students will be able to diagram the input–processing–output cycle and apply it to a real data-processing task.

## Content stub

- **Input:** collect data: forms, sensors, APIs, files, user actions — validate before accepting
- **Processing:** transform data: clean, filter, aggregate, compute; this is where value is added
- **Output:** deliver results: reports, dashboards, files, API responses, user-facing predictions
- **Storage:** between stages data is saved; databases, files, caches; affects cost and latency
- **Feedback loop:** outputs often inform the next cycle (e.g., predictions drive decisions that generate new data)

## Assessment stub

- Diagram: draw the input-process-output-feedback cycle for a weather-forecast service
- Apply: walk through the data processing cycle for a food-delivery app's ETA prediction
- Short answer: why does every processing step risk introducing errors?
