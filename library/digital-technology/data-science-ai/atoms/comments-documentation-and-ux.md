## Learning objective

Students will be able to write useful comments and docstrings, follow naming conventions, and consider user experience when designing program output.

## Content stub

- **When to comment:** explain WHY, not WHAT; flag gotchas; document unusual decisions; don't restate obvious code
- **Docstrings:** the first string inside a function — describes purpose, args, return; readable with help()
- **Naming conventions:** snake_case for variables/functions, PascalCase for classes; descriptive > short
- **User experience in CLI programs:** label output, format numbers, handle errors gracefully, don't crash on empty input
- **Readability:** short functions, consistent indentation, no magic numbers (use named constants)

## Assessment stub

- Rewrite a given 20-line function to add docstring, useful comments, and better names
- Critique: given a CLI program's output, suggest 3 UX improvements
- Short answer: when is a comment harmful rather than helpful?
