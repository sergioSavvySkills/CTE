## Learning objective

Students will be able to define an algorithm, identify its three fundamental building blocks, and express an algorithm as a flowchart or pseudocode.

## Content stub

- **What is an algorithm:** a finite, ordered sequence of steps that transforms input into output — recipes and driving directions qualify
- **Sequence:** steps execute one after another in order; the default control flow
- **Selection:** if/else branching — the algorithm chooses different steps based on a condition
- **Iteration:** loops — repeat steps while a condition holds or for a fixed count
- **Flowcharts and pseudocode:** oval = start/end, rectangle = process, diamond = decision; pseudocode is informal code-like English
- **Everyday algorithms:** brushing teeth, making a sandwich, sorting a deck of cards — same building blocks, different domain

## Assessment stub

- Write an algorithm in pseudocode for making tea, using sequence + selection + iteration
- Draw a flowchart for logging into a website with 3 attempts allowed
- Short answer: why are these three building blocks sufficient for any algorithm?
