## Learning objective

Students will be able to modify an existing algorithm, create an algorithm for a new problem, and evaluate algorithms analytically and empirically.

## Content stub

- **Modifying an algorithm:** change a step, reorder steps, add a branch — test that behavior still matches the goal
- **Creating an algorithm from a problem:** restate the problem; list inputs/outputs; break into steps; test on examples
- **Analytical evaluation:** does it always produce the correct output? does it always terminate? time/space cost?
- **Empirical evaluation:** run on test cases — normal input, boundary cases (empty, max size), invalid input
- **Comparing two algorithms:** same correctness, different costs? pick the simpler one unless performance demands otherwise

## Assessment stub

- Given an algorithm for finding the max of a list, modify it to find the min instead
- Design an algorithm to check if a word is a palindrome; test on 5 cases
- Short answer: why test boundary cases even if the normal case works?
