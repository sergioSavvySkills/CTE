## Learning objective

Students will be able to build a working prototype of an AI solution using appropriate tools.

## Content stub

- **Minimum viable prototype:** what's the smallest version that proves the concept works? cut scope ruthlessly
- **No-code prototype first:** use Teachable Machine, Lobe, or similar to validate the AI idea before writing code
- **Code prototype:** then wrap in a simple Streamlit or Gradio interface for user testing
- **Tool choices:** Streamlit for quick dashboards; Gradio for ML demos; Flask for full web apps
- **Model serving:** for class projects, CPU inference is fine; only reach for GPU when latency becomes a blocker
- **Demo video:** a 2-minute screen recording is more persuasive than 20 pages of docs

## Assessment stub

- Lab: build a Streamlit or Gradio app wrapping a trained model (use Teachable Machine output)
- Record: 2-minute demo video with voiceover
- Short answer: why is 'launch an ugly thing' better than 'plan a beautiful thing you never ship'?
