## Learning objective

Students will be able to identify physical, logical, and operational security measures, and explain how layered security (defense in depth) reduces risk.

## Content stub

- **Defense in depth:** no single control is sufficient; layering physical, network, host, and application controls
- **Physical security controls:** locks, badge access, cameras, biometric entry, security guards, man-traps
- **Network security controls:** firewalls, IDS/IPS, NAC, segmentation, DMZ; controlling what traffic flows where
- **Host security controls:** antivirus, host firewall, EDR, patching, application whitelisting
- **Identity security controls:** MFA, PAM (privileged access management), zero trust, conditional access
- **Operational controls:** policies, security awareness training, change management, incident response plans

## Assessment stub

- Defense-in-depth diagram: draw a layered security model for a small business and label each control layer
- Control classification: sort 15 security controls into Physical / Network / Host / Identity / Operational
- Short answer: why is defense in depth more effective than relying on a single strong perimeter firewall?
