## Learning objective

Students will be able to describe the purpose of an operating system, identify the major OS families, and explain the role of the kernel, file system, and user interface.

## Content stub

- **What an OS does:** manages hardware resources; provides services to applications; isolates processes from each other
- **Kernel:** core OS code running in privileged mode; hardware abstraction layer; system calls
- **OS families:** Windows, macOS, Linux distributions, ChromeOS, iOS/Android — market share and use cases
- **File system role:** organizes files into directories; tracks storage allocation; metadata (permissions, timestamps)
- **User interfaces:** CLI vs. GUI; desktop environments (Windows Explorer, GNOME, Aqua); shell as a text interface
- **Virtualization relationship:** how a hypervisor lets multiple OS instances share one hardware platform

## Assessment stub

- Comparison table: list Windows, macOS, and a Linux distro side-by-side on license, default shell, and primary use case
- Short answer: what would happen to running applications if the kernel crashed?
- Discussion: why might a company run Linux on its servers but Windows on its desktops?
